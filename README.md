# SQL Student Enrollment Analysis Project

## Overview

This project is a SQL-based data analysis case study on student enrollments, payments, and course performance. It uses a real-world dataset of Indian students to practice SQL concepts such as data cleaning, joins, aggregations, and analytical queries.

The goal of this project is to simulate a real data analyst workflow using SQL.

---

```bash
│
├── data/
│   └── Indian_Students_Data.csv
│
├── sql/
│   └── project.sql
│
└── README.md
```

## Dataset Information

The dataset contains student enrollment and payment information including:

* Student details such as name, email, and state
* Course information
* Payment status and transaction details
* Pricing information

## Key SQL Concepts Practiced

### Data Definition (DDL)

* Created tables using CREATE TABLE
* Applied constraints including PRIMARY KEY, UNIQUE, NOT NULL, and CHECK

---

### Data Loading

* Imported CSV data using BULK INSERT

---

### Data Manipulation (DML)

* Inserted new records
* Updated payment statuses
* Deleted invalid or null records

---

### Data Analysis (DQL)

Filtering and Sorting

* Used WHERE, BETWEEN, IN
* Used ORDER BY for sorting results

String Functions

* Used UPPER, LOWER, LEN, LEFT, SUBSTRING
* Extracted email domains using CHARINDEX

Aggregate Functions

* Used COUNT, SUM, AVG, MIN, MAX
* Grouped data using GROUP BY
* Filtered grouped data using HAVING

---

### Joins

* INNER JOIN between students and courses tables
* LEFT JOIN and RIGHT JOIN for unmatched records

---

### Set Operations

* UNION
* UNION ALL
* EXCEPT

---

### Advanced Querying

* Subqueries for filtering above average values
* Conditional filtering using LIKE
* Multi-condition analysis for insights

---

## Key  Insights

* Identified top performing courses based on revenue
* Analyzed student distribution across different states
* Studied payment behavior patterns
* Filtered high value students based on pricing conditions
